# เอกสารโปรเจค (ภาษาไทย)

สรุปสั้น ๆ
- โปรเจคนี้เป็นเว็บแอปตัวอย่างเขียนด้วย .NET ที่ใช้สำหรับแสดงภาพหรือทดสอบการประมวลผลภาพ (ชื่อโปรเจค: `azure-dotnet-imgweb-repo`).
- จุดประสงค์หลักคือเป็นต้นแบบสำหรับการพัฒนาและทดสอบฟีเจอร์เกี่ยวกับภาพ เช่น แสดงภาพหรือเรียกใช้งานฟังก์ชันพื้นฐาน

สิ่งที่โปรเจคทำได้ (สั้น ๆ และไม่ใช้ศัพท์เทคนิคมาก)
- เปิดเว็บหน้าเดียวเพื่อดูตัวอย่างการทำงาน
- มีโค้ดตัวอย่างสำหรับการคำนวณ/ทดสอบเล็ก ๆ (ไฟล์ `Calculator.cs` และชุดทดสอบในโฟลเดอร์ `tests`)

โครงสร้างไฟล์สำคัญ (อธิบายเป็นภาษาง่าย)
- `Program.cs` — จุดเริ่มต้นของโปรแกรม
- `Calculator.cs` — ตัวอย่างฟังก์ชันสำหรับทดสอบการคำนวณ
- `Pages/Index.cshtml` — หน้าเว็บหลักที่แสดงผล
- `appsettings.json` — การตั้งค่าพื้นฐานของโปรเจค
- `tests/` — มีตัวอย่างชุดทดสอบเพื่อยืนยันว่าโค้ดบางส่วนทำงานถูกต้อง

วิธีเริ่มใช้งาน (แบบง่าย ๆ)
- สิ่งที่ต้องมีก่อน: คอมพิวเตอร์ที่ติดตั้ง .NET (แนะนำเวอร์ชัน 8)
- คำสั่งแบบรวบรัด (พิมพ์ในเทอร์มินัล):
  - ดาวน์โหลด/เปิดโปรเจคในเครื่อง
  - รันโปรเจค: `dotnet run` (จากโฟลเดอร์โปรเจค)
  - รันชุดทดสอบ: `dotnet test` (จากโฟลเดอร์ `tests`)

วิธีเริ่มใช้งาน (แบบละเอียด)
- ติดตั้ง .NET SDK: ดาวน์โหลดจาก https://dotnet.microsoft.com ถ้ายังไม่มี (แนะนำ .NET 8)
- เปิด PowerShell แล้วไปที่โฟลเดอร์โปรเจค (ที่มีไฟล์ `Web.csproj`) เช่น:

```powershell
cd D:\2025_openai\azure-dotnet-imgweb-repo
```

- สร้างและคอมไพล์โปรเจค:

```powershell
dotnet build
```

- เรียกใช้งานแอป (จะเห็น URL ที่โปรแกรมบอก เช่น `http://localhost:5000` หรือ URL อื่นที่ระบบแสดง):

```powershell
dotnet run
```

- เปิดเว็บเบราว์เซอร์ไปที่ URL ที่ปรากฏในเทอร์มินัล

การรันชุดทดสอบ (Unit Tests)
- รันทดสอบทั้งหมด:

```powershell
dotnet test
```

- ระบุโปรเจคทดสอบโดยตรง (ถ้าต้องการ):

```powershell
dotnet test .\tests\Web.Tests\Web.Tests.csproj
```

- รันเทสต์เดี่ยว (ตัวอย่าง — ชื่อเทสต์ `Divide_ZeroByZero_ReturnsNaN` ที่เพิ่มเข้ามา):

```powershell
dotnet test --filter "FullyQualifiedName~Divide_ZeroByZero_ReturnsNaN"
```

หมายเหตุการรันทดสอบ
- ถ้ารัน `dotnet run` แล้วไม่เห็นหน้าเว็บ ให้ตรวจข้อความในเทอร์มินัลว่ามีข้อผิดพลาดอะไร (เช่น พอร์ตใช้แล้วหรือปัญหาในการคอมไพล์)
- ถ้าชุดทดสอบล้มเหลว ให้ดูข้อความที่ `dotnet test` แสดงไว้ (ชื่อ test และแทรซแบ็ก) แล้วส่งข้อความมาหาผมได้ ผมจะช่วยวิเคราะห์

คำแนะนำสำหรับผู้ที่ไม่ชำนาญ
- ถ้าไม่แน่ใจว่าจะติดตั้ง .NET ยังไง ให้ค้นหา "Install .NET" จากเว็บของ Microsoft และเลือกเวอร์ชันที่แนะนำ
- ถ้าเปิดโปรเจคแล้วไม่แสดงผล ให้ลองดูข้อความผิดพลาดที่ขึ้นในเทอร์มินัล แล้วแชร์ข้อความนั้นเพื่อให้ช่วยแก้ปัญหาได้

- ข้อควรระวัง
- ไฟล์ในโฟลเดอร์ `bin/` และ `obj/` เป็นไฟล์ที่สร้างขึ้นอัตโนมัติจากการคอมไพล์ ไม่จำเป็นต้องแก้ไข
